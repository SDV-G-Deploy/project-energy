# UX + A11Y CHECKLIST (V1)

Дата проверки: 2026-03-23

---

## 1) Контраст

- [x] Основной текст на светлом фоне имеет высокий контраст (тёмный текст + белые/светлые карточки).
- [x] Secondary text не уходит в «бледно-серый» ниже читабельного порога.
- [x] Состояния кнопок (default/hover/active/focus) визуально различимы.

Принятое решение:
- Использован контрастный набор цветов с акцентом и отдельным цветом focus-ring.

---

## 2) Размер шрифта на мобиле

- [x] Базовый шрифт не ниже 16px.
- [x] Заголовки масштабируются через `clamp()`.
- [x] Межстрочный интервал повышен для коротких экранов.

Принятое решение:
- Mobile-first сетка, карточки и текст оптимизированы под чтение с руки.

---

## 3) Focus-visible и клавиатурная навигация

- [x] Для интерактивных элементов есть `:focus-visible`.
- [x] Focus ring заметен и не перекрывается стилями.
- [x] Элементы имеют логичный порядок tab-навигации.

Принятое решение:
- Единый фокус-стиль для кнопок/ссылок/inputs.

---

## 4) Reduced motion

- [x] В `prefers-reduced-motion: reduce` отключены/упрощены анимации.
- [x] Сглаженный скролл не обязателен для функциональности.
- [x] Нет критически важной информации, завязанной только на анимацию.

Принятое решение:
- Все анимации короткие, optional, и отключаемые.

---

## 5) Tap targets и мобильные взаимодействия

- [x] Основные кнопки крупные (удобные для большого пальца).
- [x] Дистанция между интерактивными элементами достаточна.
- [x] Состояния «нажал/выполнено/сохранено» видны мгновенно.

Принятое решение:
- Минимальная высота кнопок ≥ 52px, крупные зоны касания.

---

## 6) Edge cases

- [x] Пустое состояние (если JSON не загрузился) → fallback-контент в JS.
- [x] Повторный вход в тот же день → показывается сохранённый прогресс.
- [x] Пропуск дней → soft re-entry copy, без shame-механик.
- [x] Повторное нажатие «выполнено» не дублирует начисления.
- [x] Повреждённый localStorage не ломает интерфейс (safe parse + defaults).

---

## 7) ARIA и живые объявления

- [x] Для динамических сообщений используется `aria-live="polite"`.
- [x] Структура секций и заголовков семантически корректна.
- [x] Слайдеры энергии имеют текстовые метки и текущие значения.

---

## 8) Что ещё проверить в следующей итерации

1. Реальный аудит WCAG с автоматизированным инструментом (Lighthouse + axe).
2. Usability-тест 5–7 пользователей на мобильном: первый клик и completion.
3. Сравнение коротких/длинных формулировок CTA (A/B, hypothesis).
4. Проверка цветовых тем для людей со сниженным контрастным восприятием.
