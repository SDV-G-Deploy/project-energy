## Что сделано в sprint-1

### 1) Outcome-first Hero
- Переписан hero под ценность «анти-прокрастинационный reset за 60 секунд».
- Primary CTA обновлён: `⚡ Запустить reset за 60 сек`.

### 2) 3 сценарных CTA
- Добавлены сценарии старта:
  - `Тяжело начать` (recovery),
  - `Расфокус` (environment),
  - `Упадок энергии` (nutrition).
- Сценарии используют текущую архитектуру `todayActionMode` и не ломают текущий flow ритуала.

### 3) Trust-блок
- Добавлен блок «Кому подходит / Не подходит».
- Добавлен короткий FAQ.
- Добавлены mini-proof элементы:
  - локальный TTFV,
  - число стартов,
  - число completion.

### 4) Post-completion блок (улучшен)
- В recap добавлены:
  - `Следующий лучший шаг` (next best action),
  - `Когда вернуться` (подсказка по времени/якорю).
- Добавлены post-action CTA:
  - `Сделать ещё один сейчас`,
  - `Выбрать когда вернуться`.

### 5) Минимальная событийная аналитика (локально)
- Унифицированы ключевые события:
  - `hero_cta_click`,
  - `ritual_start`,
  - `ritual_complete`,
  - `post_action_click`.
- Схема событий задокументирована в `README.md`.

### 6) Обратная совместимость
- Основная механика ритуалов/таймера/трекера сохранена.
- Исторические completion-события поддержаны в weekly-агрегации (`mini_ritual_complete` + `ritual_complete`).
